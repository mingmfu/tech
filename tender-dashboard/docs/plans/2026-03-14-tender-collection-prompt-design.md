## Step 5: 智能推荐

基于多维度评估，为每个项目计算得分并输出推荐报告。

### 5.1 评分维度（满分100分）

对每个项目进行6个维度的评估：

#### 1. 金额匹配度（20分）

**评分标准**：
- 18-20分：10-50万元（理想区间，工作量和收益平衡）
- 12-17分：50-100万元（略大，但可行）
- 5-11分：<10万元（工作量小但利润低）或 >100万元（竞争激烈，难度大）
- 0-4分：金额未明确或异常

**说明**：10-50万元的项目通常竞争适中，既能保证收益又不会过于复杂。

#### 2. 时间窗口（20分）

**评分标准**（基于距离截止的天数）：
- 18-20分：10-20天（准备时间充足，又不至于夜长梦多）
- 12-17分：5-10天（略紧张，但可接受）
- 5-11分：20-30天（时间过长，可能有变数）
- 0-4分：<5天（时间太紧）或 >30天（过久）

**说明**：10-20天的时间窗口既能充分准备标书，又不会因为时间过长导致市场变化。

#### 3. 资质匹配（20分）

**评分标准**：
- 18-20分：完全匹配，所有要求均可满足
- 12-17分：基本匹配，个别次要要求可能不满足
- 5-11分：部分匹配，主要要求可达成但存在短板
- 0-4分：不匹配，关键资质或经验缺失

**评估要点**：
- 是否具备会计师事务所执业资格
- 是否有足够的注册会计师
- 是否有类似行业审计经验
- 是否满足业绩要求（项目数量、规模）

#### 4. 历史中标率（15分）

**评分标准**：
- 13-15分：该类型/该业主历史中标率高（>50%）
- 8-12分：历史中标率中等（20-50%）
- 3-7分：历史中标率低（<20%）
- 0-2分：首次接触该类型/业主，无历史数据

**评估要点**（基于已采集项目推断）：
- 业主是否为经常招标的单位
- 该类型项目是否竞争激烈
- 是否有明显的"内定"迹象

#### 5. 竞争程度（15分）

**评分标准**：
- 13-15分：竞争程度低，预估投标人数<5家
- 8-12分：竞争程度中等，预估5-10家
- 3-7分：竞争激烈，预估>10家
- 0-2分：竞争异常激烈或有明显优势方

**评估要点**：
- 项目金额大小（金额越大，竞争越激烈）
- 项目类型通用性（通用服务竞争大）
- 资质门槛高低（门槛低则竞争激烈）
- 业主知名度（知名业主吸引更多投标人）

#### 6. 客户价值（10分）

**评分标准**：
- 8-10分：央企、国企总部、知名上市公司
- 5-7分：地方国企、一般上市公司
- 2-4分：民营企业、事业单位
- 0-1分：其他

**评估要点**：
- 客户性质（国企/上市公司稳定性高）
- 客户规模（大型企业后续合作机会多）
- 付款信誉（国企通常付款及时）

### 5.2 计算总分

**公式**：
```
总分 = 金额匹配度 + 时间窗口 + 资质匹配 + 历史中标率 + 竞争程度 + 客户价值
```

# Evaluation Criteria Detail

## 评分算法伪代码

```
function calculateScore(project):
    score = {}
    
    # 1. 金额匹配度 (20分)
    amount = parseFloat(project.项目金额)
    if 10 <= amount <= 50:
        score.金额匹配度 = 18-20
    elif 50 < amount <= 100 or 5 <= amount < 10:
        score.金额匹配度 = 12-17
    else:
        score.金额匹配度 = 0-11
    
    # 2. 时间窗口 (20分)
    daysLeft = calculateDays(project.截止时间, now())
    if 10 <= daysLeft <= 20:
        score.时间窗口 = 18-20
    elif 5 <= daysLeft < 10:
        score.时间窗口 = 12-17
    else:
        score.时间窗口 = 0-11
    
    # 3. 资质匹配 (20分)
    # 需人工判断或基于历史数据
    score.资质匹配 = 人工评估结果
    
    # 4. 历史中标率 (15分)
    # 基于项目数量和业主类型推断
    score.历史中标率 = 推断结果
    
    # 5. 竞争程度 (15分)
    # 基于金额、资质门槛推断
    if project.项目金额 > 100:
        score.竞争程度 = 0-7
    elif project.项目金额 < 10:
        score.竞争程度 = 8-12
    else:
        score.竞争程度 = 13-15
    
    # 6. 客户价值 (10分)
    if "国企" in project.招标单位 or "集团" in project.招标单位:
        score.客户价值 = 8-10
    elif "公司" in project.招标单位:
        score.客户价值 = 2-7
    else:
        score.客户价值 = 0-2
    
    return sum(score.values()), score
```

# Constraints

## 执行约束

1. **验证码处理**：
   - 如遇图形验证码，停止操作
   - 报告："遇到验证码，无法自动登录，需要人工处理"

2. **超时处理**：
   - 单页加载超时：30秒未完成加载则跳过
   - 报告："项目[项目名称]加载超时，已跳过"

3. **日期检查**：
   - 只采集**未截止**的项目
   - 如项目已截止（当前日期 > 截止时间），跳过
   - 报告："项目[项目名称]已截止，已跳过"

4. **地区检查**：
   - 确保项目地点在安徽省内
   - 如地点不在安徽，跳过
   - 报告："项目[项目名称]不在安徽地区，已跳过"

5. **数据完整性**：
   - 如关键字段缺失过多（如缺少项目名称、招标单位、截止时间），标记为"部分提取"
   - 尽可能提取已有字段，不因此跳过整个项目

6. **隐私保护**：
   - 联系人电话/邮箱可部分隐藏（如：138****1234）
   - 不要存储或输出完整的敏感信息

7. **异常报告**：
   - 遇到网站改版、字段位置变更等异常，详细报告问题
   - 如无法定位某个字段，标记为"未找到"并继续

## 输出约束

1. **JSON格式**：
   - 必须输出有效的JSON格式
   - 所有字符串值使用双引号
   - 中文字段名使用中文，不使用英文转写

2. **编码**：
   - 使用UTF-8编码
   - 确保中文正常显示

3. **文件保存**：
   - 将完整JSON保存到 `tender_collection_[日期时间].json`
   - 如用户指定了文件名，使用用户指定的名称

## 性能约束

1. **并发控制**：
   - 不要同时打开多个详情页
   - 按顺序逐个处理项目

2. **请求频率**：
   - 页面间切换间隔至少2秒
   - 避免对服务器造成过大压力

3. **采集数量**：
   - 如无特殊说明，采集筛选结果中的所有项目
   - 如项目数量>50，可先采集前20个，询问用户是否继续

# Error Handling

## 常见错误及处理

| 错误场景 | 处理方案 | 报告内容 |
|----------|----------|----------|
| 无法访问网站 | 检查网络连接，重试1次后停止 | "无法访问合信招标网，请检查网络" |
| 登录失败 | 检查账号密码，停止 | "登录失败：账号或密码错误" |
| 遇到验证码 | 停止，请求人工处理 | "遇到验证码，无法自动登录" |
| 页面结构变化 | 尝试多种选择器，如失败则报告 | "页面结构变化，无法定位字段" |
| 项目加载超时 | 跳过该项目 | "项目[名称]加载超时，已跳过" |
| 项目已截止 | 跳过该项目 | "项目[名称]已截止，已跳过" |
| 地区不匹配 | 跳过该项目 | "项目[名称]不在安徽地区，已跳过" |
| 关键字段缺失 | 标记为"部分提取"，继续 | "项目[名称]部分字段缺失" |
| JSON解析错误 | 检查提取逻辑，修复后重试 | "数据格式化错误" |

## 错误恢复策略

1. **单个项目失败**：
   - 不影响其他项目的采集
   - 记录失败原因，继续处理下一个

2. **网络中断**：
   - 等待10秒后重试
   - 重试2次仍失败则停止

3. **网站改版**：
   - 保存当前已采集的数据
   - 详细报告页面变化
   - 请求人工介入更新提取规则
